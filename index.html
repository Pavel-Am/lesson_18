Общие сведения

CSS Grid – это техника в каскадных таблицах стилей, позволяющая веб-разработчикам создавать сложные адаптивные макеты веб-дизайна более легко и последовательно в различных браузерах. Есть и другие методы макета вебстраниц, которые использовались ранее, включая tables, CSS Box model и CSS Flexbox.

CSS Grid – это модель шаблона, оптимизированная для двумерных шаблонов. Используется в качестве модели шаблонов вебсайтов, форм, галерей и всего, что требует точного и внимательного позиционирования. Сети CSS – это новый подход к созданию адаптивных сайтов с большим количеством блоков, расположенных в любых местах сайта. Кроме CSS Grid существует также технология Flexbox, которая очень схожа с CSS Grid.

Grid модуль в CSS был разработан CSS Working Group для того, чтобы предоставить лучший способ создания шаблонов в CSS. Он попал в W3C Candidate Recommendation в феврале 2017 года, а основные браузеры начали его поддержку в марте 2017 года. Используется для разработки front-end. Данная технология обладает мощным функционалом и интуитивно понятным синтаксисом, а шаблоны на grid несомненно будут изменять то, на чем создается Web.

CSS Grid можно назвать улучшенной версией Flexbox, ведь Flexbox позволяет работать только в одной плоскости: либо создавать столбцы, либо создавать строчки. CSS Grid позволяет производить больше, так как данная технология работает в обеих плоскостях одновременно. Таким образом создание адаптивных блоков сайта происходит проще, а возможности для расположения объектов увеличиваются.

По состоянию на февраль 2023 года большинство браузеров поставляли беспрефиксную поддержку CSS Grid 96.36%: Chrome (включая Android), Firefox, Safari (включая iOS) и Opera. Internet Explorer 10 и 11, с другой стороны, поддерживают его старую реализацию с устаревшим синтаксисом. Все современные браузеры для мобильных устройств поддерживают CSS Grid кроме браузеров Opera mini.

grid-template-columns
grid-template-rows

Явная сетка в гридах формируется из колнок и рядов для которых заданны конкретные размеры. Для формирования колонок мы используем свойство grid-template-columns, а для формирования рядов - свойство grid-template-rows. В значении свойств grid-template-columns нам необходимо указывать сразу ширины колонок через пробел и сколько мы ширин укажем столько колонок явной сетки у нас и будет.

Для того, чтобы построить трех-колоночную систему необходимо для свойства grid-template-columns задавать не фиксированные или процентные значения, а отдельное значение, выведенное специально для грид - fr. Которое означает, что колонка, для которой задали значение в fr будет стремиться занять все свободное пространство. Таким образом задавая трем колонкам значение в 1fr все три колонки будут стремиться занять все свободное пространство тем самым формируя одинаковые по ширине колонки.

Что касается свойства grid-template-rows, то для него задаются значения по такому же принципу с одним лишь отличаем в том, что для рядов задается не ширина, а высота. Ряды, высота которых не была указана в свойстве grid-template-rows формируют уже не явную сетку, которую можно регулировать другими свойствами.

Определяют местоположение в сетке ссылаясь на конкретные линии. grid-column-start / grid-row-start - это линия с которой начинается элемент, а grid-column-end / grid-row-end - это линия на которой элемент заканчивается. order

Свойство order можно использовать для указания порядка, в котором различные элементы должны быть размещены внутри сетки. По умолчанию элементы размещаются в том порядке, в котором они появляются в DOM. Например, если элемент A находится выше элемента B в фактическом исходном документе, он также будет помещен в сетку перед элементом B. В зависимости от вашего проекта это может быть или не быть желаемым поведением.

Это orderсвойство может быть очень полезным, особенно когда элементов много или элементы добавляются динамически. В таких случаях, если вы хотите, чтобы элемент всегда помещался в конец сетки, вы можете легко сделать это с помощью свойства order.

Элементы с наименьшей стоимостью заказа размещаются в сетке первыми. Элементы с более высокими значениями размещаются позже. Элементы с одинаковым значением порядка будут размещены в том порядке, в котором они появляются в исходном документе.
